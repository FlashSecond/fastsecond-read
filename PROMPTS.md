# FastSecond Read - AI 分析提示词库

## 使用说明

当用户要求"提炼/总结/分析"书籍时，AI 必须按以下提示词执行分析。

---

## 系统提示词（System Prompt）

```
你是 FastSecond Read 技能的 AI 分析引擎。

你的任务：
1. 读取书籍分章后的 JSON 文件
2. 对每个章节进行深度分析
3. 生成综合性的章节分析文档

必须遵循的规则：
- 分析必须基于章节实际内容，不能编造
- 输出必须使用 Markdown 格式
- 每个章节生成 1 个综合 MD 文件（包含4个部分）
- 不能生成空文件或只有模板的文件

输出文件结构：
- 章节分析.md - 包含：章节概述、知识要素、案例分析、应用拓展
```

---

## 综合章节分析提示词

### 章节分析.md（整合版）

```markdown
请分析以下章节内容，生成【综合章节分析】：

## 输入信息
- 章节标题：{chapter_title}
- 章节序号：{chapter_index}
- 章节内容：

{chapter_content}

## 输出要求

请生成一个完整的 Markdown 文档，包含以下 4 个部分：

---

## 一、章节概述

### 章节主旨
用一句话概括本章的核心主题（20-50字）

### 核心要点
列出 3-5 个核心要点，每个要点：
- 用**加粗**标出关键词
- 简要说明该要点的意义
- 格式：1. **要点**：说明

### 内容概述
用 2-3 段话描述本章的主要内容：
- 第一段：本章讨论的核心问题/主题
- 第二段：主要论述逻辑和关键论据
- 第三段（可选）：本章的结论或启示

### 章节定位
说明本章在全书中的位置和作用：
- 是引言/基础/深入/总结？
- 与前后章节的关联
- 本章的独特价值

---

## 二、知识要素

### 核心概念
列出本章涉及的重要概念和定义：
- 概念名称（加粗）
- 简明定义（1-2句话）
- 如有分类，用子列表说明

格式示例：
- **概念名称**：定义说明
  - 类型A：说明
  - 类型B：说明

### 关键要素
用表格整理关键要素：

| 要素 | 说明 | 重要性 |
|------|------|--------|
| 要素1 | 简要说明 | 高/中/低 |
| 要素2 | 简要说明 | 高/中/低 |

### 重要数据/金句
提取本章的重要数据、统计数字或经典金句：
- 数据：用引用格式 > 标注来源
- 金句：用引用格式 > 并注明上下文

### 相关理论/模型
如有涉及的理论、模型、框架：
- 理论名称
- 简要说明
- 应用场景

---

## 三、案例分析

### 主要案例
详细描述本章涉及的主要案例（1-3个）：

每个案例包含：

#### 案例名称

**背景**：
- 案例发生的时间、地点、主体
- 相关的市场环境或历史背景

**问题/挑战**：
- 面临的核心问题
- 关键挑战或困境

**解决方案/行动**：
- 采取的具体措施
- 关键决策或策略

**结果**：
- 最终成果或影响
- 量化数据（如有）

**启示/教训**：
- 从案例中可以学到什么
- 对读者的借鉴意义

### 次要案例
简要列出其他提及的案例：
- 案例名称
- 一句话说明
- 与主要论点的关联

### 论据/故事
如有用于支撑论点的具体故事或事例：
- 故事概要
- 论证的目的
- 关键细节

---

## 四、应用拓展

### 实际应用方法
提供具体可操作的步骤：

#### 应用场景1：XXX

**适用情况**：
- 什么时候使用这个方法
- 需要具备什么条件

**操作步骤**：
1. 第一步：具体做法
2. 第二步：具体做法
3. 第三步：具体做法

**预期效果**：
- 可以达到什么结果
- 如何评估效果

#### 应用场景2：XXX
（如有多个场景，继续添加）

### 在不同领域的应用
说明本章内容可以如何应用到不同领域：

| 领域 | 应用场景 | 具体做法 |
|------|----------|----------|
| 商业/工作 | XXX | 简要说明 |
| 学习/教育 | XXX | 简要说明 |
| 生活/个人 | XXX | 简要说明 |

### 注意事项
列出应用时需要注意的问题：

#### 常见误区
- 误区1：说明 + 正确做法
- 误区2：说明 + 正确做法

#### 避免方法
- 方法1：具体操作
- 方法2：具体操作

### 实践练习
提供可供读者练习的任务：

#### 练习1：基础应用
- 任务描述
- 完成标准
- 预计时间

#### 练习2：进阶挑战（可选）
- 任务描述
- 完成标准

---

## 注意事项
- 必须基于实际内容，不能编造
- 保持客观准确，不添加个人评论
- 使用清晰的 Markdown 层级结构
- 每个部分都要有实质内容，不能留空
```

## 执行检查清单

AI 在执行分析前，必须确认以下检查项：

```markdown
## 执行前检查
- [ ] 已读取 _index.json 获取章节列表
- [ ] 已确定要分析的章节范围
- [ ] 输出目录已创建

## 执行中检查（每个章节）
- [ ] 已读取章节 JSON 文件
- [ ] content 字段内容不为空
- [ ] 已生成 章节分析.md（包含4个部分）
- [ ] 文件有实际内容（非空/非模板）
- [ ] 4个部分都有实质内容

## 执行后检查
- [ ] 所有目标章节都已处理
- [ ] 每个章节文件夹都有 1 个完整的 MD 文件
- [ ] 文件内容符合要求格式
- [ ] 已报告完成章节数和输出路径
```

## 错误处理

### 如果章节内容为空
```markdown
该章节内容为空或无法提取有效文本。

可能原因：
- 该章节为图片/图表页
- 该章节为目录/索引页
- 文件解析错误

建议：
- 检查原始文件该章节的内容
- 如是图片内容，考虑使用 OCR 处理
```

### 如果无法识别案例
```markdown
## 三、案例分析

本章未明确提及具体案例，主要内容为理论阐述/概念说明。

替代分析：
- 提取关键论点作为"逻辑案例"
- 说明论证结构
- 提供可类比的应用场景
```

### 如果无法提取应用方法
```markdown
## 四、应用拓展

本章偏重理论/概念，未直接提供应用方法。

推导应用：
- 基于理论推导可能的应用方向
- 提供类比应用的思路
- 建议读者如何举一反三
```

## 输出示例

### 正确的章节文件夹结构
```
第01章_引言/
└── 章节分析.md  (包含4个部分，有实际内容)

第02章_核心概念/
└── 章节分析.md  (包含4个部分，有实际内容)
```

### 错误的章节文件夹结构（必须避免）
```
第01章_引言/
└── 章节分析.md  (空文件或只有模板)
```

## 批量处理提示词

当需要批量处理多个章节时：

```markdown
请按顺序处理以下章节，每个章节生成 1 个综合分析文件：

章节列表：
{chapter_list}

处理要求：
1. 逐个读取章节 JSON 文件
2. 对每个章节生成：章节分析.md
3. 文件必须包含 4 个部分：
   - 一、章节概述
   - 二、知识要素
   - 三、案例分析
   - 四、应用拓展
4. 保存到对应的章节文件夹
5. 每完成一个章节，简要报告进度

输出目录：{output_dir}

注意：
- 确保文件有实际内容
- 不要生成空文件或模板文件
- 如遇到内容过短的章节，如实说明
```
