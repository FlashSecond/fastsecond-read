# FastSecond Read - 分析输出模板库

本文档定义AI分析章节内容时的输出格式模板。

---

## 输出模式

### 模式一：按章节聚合（默认）

每个章节独立成册，适合深入阅读单个章节。

```
书籍总结/
├── 第01章_标题/
│   ├── 01_章节概述.md
│   ├── 02_知识要素.md
│   ├── 03_案例分析.md
│   └── 04_应用拓展.md
├── 第02章_标题/
│   └── ...
└── 00_全书总结.md
```

**适用场景**："提炼第3章"、"分析第5-10章"

### 模式二：按类型聚合

同类型内容集中呈现，适合横向对比全书结构。

```
书籍总结/
├── 01_全书章节概述.md      # 所有章节的概述
├── 02_全书知识要素.md      # 所有章节的知识要素
├── 03_全书案例分析.md      # 所有章节的案例分析
├── 04_全书应用拓展.md      # 所有章节的实践方法
└── 00_全书总结.md
```

**适用场景**："提炼这本书"、"总结全书"、"提取全书的知识要素"

## 模式一：按章节聚合模板

### 01_章节概述.md（单章）

```markdown
# 第X章：章节标题

## 章节主旨
用1-2句话概括本章核心主题和作者意图。

## 章节概述
详细描述本章的主要内容结构，包括：
- 开篇引入的方式和背景
- 主要论述的逻辑脉络
- 各小节的核心观点
- 结尾的总结或过渡

## 核心要点
1. **要点一**：具体阐述
2. **要点二**：具体阐述
3. **要点三**：具体阐述
```

### 02_知识要素.md（单章）

```markdown
# 第X章：章节标题 - 知识要素

## 核心概念

### 概念名称
**定义**：简明准确的定义
**解释**：详细说明和背景
**重要性**：为什么这个概念重要

## 关键术语
- **术语1**：解释说明
- **术语2**：解释说明

## 金句摘录
> "原文引用内容" —— 出处/语境说明

## 重要数据/事实
- 数据1：说明
- 数据2：说明
```

### 03_案例分析.md（单章）

```markdown
# 第X章：章节标题 - 案例分析

## 案例一：案例名称

### 背景
案例发生的背景和情境

### 经过
详细描述事件发展过程

### 结果
最终 outcome 和影响

### 启示
从案例中提炼的教训或原理

## 案例二：案例名称
...
```

### 04_应用拓展.md（单章）

```markdown
# 第X章：章节标题 - 应用拓展

## 实践方法

### 方法名称
**适用场景**：什么时候使用
**操作步骤**：
1. 步骤一
2. 步骤二
3. 步骤三
**注意事项**：关键提醒

## 工具/资源推荐
- **工具/资源名称**：简要说明和用途

## 延伸思考
1. 与本章内容相关的思考问题
2. 可以进一步探索的方向

## 行动建议
- [ ] 具体可执行的行动项
```

## 00_全书总结.md（可选）

当用户要求整书总结时，在章节分析完成后生成：

```markdown
# 《书名》全书总结

## 书籍概览
- **作者**：
- **核心主题**：
- **章节数**：X章
- **总字数**：约X万字

## 内容架构
```
第1章：XXX —— 主题简述
第2章：XXX —— 主题简述
...
```

## 核心论点
1. **论点一**：阐述
2. **论点二**：阐述
3. **论点三**：阐述

## 知识地图
- **概念A**：关联章节
- **概念B**：关联章节

## 实践价值
- 本书对XX人群的价值
- 可立即应用的方法
- 需要深入思考的内容

## 阅读建议
- 重点章节：第X章、第X章
- 快速浏览：第X章
- 反复研读：第X章
```

## 模式选择规则

| 用户指令 | 输出模式 | 说明 |
|---------|---------|------|
| "提炼这本书" | 模式二（按类型聚合） | 全书横向对比 |
| "总结全书" | 模式二（按类型聚合） | 全书横向对比 |
| "提炼第3章" | 模式一（按章节聚合） | 单章深入分析 |
| "分析第5-10章" | 模式一（按章节聚合） | 指定章节深入 |
| "提取全书的知识要素" | 模式二（仅02文件） | 只生成知识要素 |

---

## 分析原则

1. **忠于原文**：基于章节JSON的实际内容进行分析，不臆测
2. **结构清晰**：使用Markdown标题层级组织内容
3. **详略得当**：重要内容详细展开，次要内容简要概括
4. **实用导向**：应用拓展部分要具体可操作
5. **格式统一**：同类输出保持一致的模板结构
