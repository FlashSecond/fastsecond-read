## 分章规则摘要

### EPUB/HTML/Markdown

#### 一级分章（默认）
- **章节标题**：h1/h2 标签 → 单独成文件
- **小标题**：h3-h6 标签 → 作为正文内容
- **正文**：归属于最近章节

#### 多级分章
- **一级章节**：h1/h2 标签 → 单独成文件
- **二级章节**：h3/h4 标签 → 单独成文件
- **三级章节**：h5/h6 标签 → 单独成文件
- **正文**：归属于最近章节

**多级分章输出示例**：
```
第01章_学习基础/
├── 01_章节概述.md
├── 02_知识要素.md
├── 03_案例分析.md
└── 04_应用拓展.md
第01-1章_什么是学习/      ← h3小节
├── 01_章节概述.md
├── 02_知识要素.md
├── 03_案例分析.md
└── 04_应用拓展.md
```

### PDF/Word
- **章节标题**：字号 > 正文 × 1.3、居中、粗体、单独成行
- **章节内小标题**：数字列表格式（如"1. 标题"）
- **正文**：靠左对齐、字体统一

## 2026-04-30 优化：解决"只分章不分析"问题

### 问题描述
发现 AI 在执行"提炼书籍"任务时，经常出现：
- 只完成了分章（生成JSON文件）
- 忘记了对每个章节进行 AI 分析
- 用户只得到了 `_index.json` 和章节JSON，没有 MD 分析文件

### 解决方案

#### 1. 更新 SKILL.md
- 添加明确的"分章后必须执行章节分析"警告
- 添加 AI 执行检查清单
- 添加常见错误示例

#### 2. 新增脚本

**chapter_analyzer.py** - 章节分析器
- 为每个章节生成 4 类分析文件
- 生成 AI 分析提示词
- 支持章节范围筛选

**auto_process.py** - 全自动处理
- 一键完成分章 + 分析全流程
- 两个步骤自动衔接
- 支持命令行参数

**verify_analysis.py** - 验证工具
- 检查分析结果完整性
- 统计完成率
- 识别空文件/模板文件

#### 3. 关键改进
- SKILL.md 中明确警告："不要只做 Step 1！必须执行 Step 2 的章节分析！"
- 添加检查清单，确保 AI 不会遗漏
- 提供验证工具，方便检查完成情况

### 使用方式

```bash
# 方式1：全自动处理（分章+分析）
python auto_process.py "书籍.epub"

# 方式2：只分章
python book_processor.py "书籍.epub"

# 方式3：只分析（JSON已存在）
python chapter_analyzer.py "书籍JSON目录"

# 方式4：验证结果
python verify_analysis.py "分析结果目录"
```

## 2026-04-30 优化：AI 分析提示词设计

### 问题
AI 分析章节时，输出质量不稳定，有时过于简单，有时偏离要求。

### 解决方案

#### 1. 创建 PROMPTS.md
设计完整的提示词模板，包含：
- **系统提示词**：定义 AI 角色和任务
- **4类分析提示词**：概述/知识/案例/应用，每类都有详细结构要求
- **执行检查清单**：确保不遗漏步骤
- **错误处理模板**：应对内容为空等异常情况
- **批量处理提示词**：多章节处理指南

#### 2. 创建 QUICKREF.md
快速参考卡片，包含：
- 一句话总结执行流程
- 可直接复制的简化提示词
- 检查清单
- 常见问题解答
- 命令速查

### 提示词设计原则

1. **结构化输出**：明确指定 Markdown 标题层级
2. **内容约束**：规定字数、格式、必须包含的要素
3. **示例引导**：提供格式示例，减少歧义
4. **变量替换**：使用 `{title}`、`{content}` 等占位符，便于程序化替换
5. **截断提示**：`content[:3000]` 避免超出上下文限制

### 文件清单

| 文件 | 用途 |
|------|------|
| PROMPTS.md | 完整提示词库，详细版 |
| QUICKREF.md | 快速参考，简化版 |

## 2026-04-30 优化：JSON → Markdown

### 变更内容

将分章输出从 **JSON 格式** 改为 **Markdown 格式**：

**旧版流程**：
```
书籍文件 → 分章 → JSON文件 → AI读取 → 分析结果MD
```

**新版流程**：
```
书籍文件 → 分章 → MD文件（排版好）→ AI读取 → 分析结果MD
```

### 变更原因

- 用户可以直接阅读分章后的MD文件
- 排版美观，体验更好
- 减少格式转换，流程更简洁

### 文件更新

- 修改 `book_processor.py` - 直接输出排版好的MD文件
- 修改 `chapter_analyzer.py` - 读取MD文件进行分析
- 修改 `auto_process.py` - 适配新的MD流程
- 修改 `verify_analysis.py` - 验证MD文件
- 修改 `SKILL.md` - 更新文档说明

### MD文件格式

```markdown
# 章节标题

---

**章节序号**：第1章  
**章节层级**：1  
**字数统计**：1500 字  
**生成时间**：2024-04-30 12:00  

---

章节正文内容...

---

*由 FastSecond Read 自动生成*
```
