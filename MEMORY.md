## 2026-05-05 更新：EPUB 读取器增强（多书合并支持）

### 问题描述
EPUB 文件包含 4 本独立书籍，但第 4 本书《行为设计学：打造峰值体验》因缺少 H1 标题被合并到第 7 章，导致整本书内容（约 17 万字）丢失在单个章节中。

### 根因分析
1. **无 H1 标题**：text00072-text00078 等关键文件没有 H1 标题，只有 H3 标题
2. **EPUB 读取器逻辑**：依赖 H1 标签识别章节边界，无 H1 时无法识别新书开始
3. **多书合并**：4 本书简单拼接，没有正确的合集结构

### 解决方案

#### 1. 检测无 H1 的异常结构
```python
# 当 HTML 文件没有 H1 时，自动提升包含新书标记的标题为 H1
if not has_h1 and headings_in_file:
    for h in headings_in_file:
        if any(indicator in h['title'] for indicator in NEW_BOOK_INDICATORS):
            h['level'] = 1  # 提升为 H1
            h['is_promoted'] = True
```

#### 2. 超大章节拆分
```python
# 当章节内容超过 10 万字符时，基于内部标题结构自动拆分
if total_chars > 100000:
    sub_chapters = self._split_large_chapter(heading, content_blocks)
```

#### 3. 避免重复处理
```python
# 拆分后跳过同一文件中的其他标题
skip_until_file = heading['file_path']
```

### 修复效果
- 章节总数：21 → 36
- 第 4 本书《行为设计学：打造峰值体验》正确分离为 16 个独立章节
- 4 本书全部正确识别和分离

### 文件变更
- 修改：`scripts/readers/epub_reader.py` - 添加无 H1 检测、超大章节拆分、重复处理避免逻辑
- 修改：`SKILL.md` - 更新 EPUB 分章规则和更新历史

## 2026-05-05 更新：PDF 读取器规则最终优化

### 问题描述
用户反馈 "（一）消费" 被识别为二级标题，但应该是三级标题。

### 根因分析
三级标题字号上限为 1.1，而 "（一）消费" 的字号比例为 1.18，落在二级范围（1.1-1.4）。

### 解决方案
统一二级/三级标题的字号范围为 **1.0-1.4**，完全通过 **x0 与正文位置关系** 区分：

```python
# 二级标题：字号 1.0-1.4 + x0 远离正文
if 1.0 <= font_ratio < 1.4 and x0_gap > 10:
    return 2

# 三级标题：字号 1.0-1.4 + x0 接近正文
if 1.0 <= font_ratio < 1.4 and x0_gap <= 10:
    return 3
```

### 判定规则总结

| 层级 | 字号条件 | x0 条件 | 核心区分点 |
|------|---------|---------|-----------|
| **一级** | ≥ 1.4 | 远离正文 (>10) | 字号门槛最高 |
| **二级** | 1.0-1.4 | 远离正文 (>10) | x0 远离正文 |
| **三级** | 1.0-1.4 | 接近正文 (≤10) | x0 接近正文 |

### 验证结果

**"（一）消费" 判定**：
- 字号比例：1.18
- x0 差距：接近正文（≤10点）
- 结果：**三级标题** ✅

**"一、消费基本概念" 判定**：
- 字号比例：1.37
- x0 差距：远离正文（>10点）
- 结果：**二级标题** ✅

### 文件变更
- 修改：`scripts/readers/pdf_reader.py` - 更新三级标题字号上限（1.1 → 1.4）
- 修改：`SKILL.md` - 更新分章规则表格

## 2026-05-05 更新：PDF 读取器规则优化

### 变更内容

#### 1. 移除三级标题模式匹配
**原逻辑**：通过正则匹配 "一、"、"1."、"（一）" 等模式强制判定三级标题

**问题**：
- 模式匹配干扰纯版面特征判断
- "一、消费基本概念" 等标题无法根据实际字号灵活判定层级

**新逻辑**：完全基于 **字号比例 + x0位置** 判断，移除所有模式匹配

#### 2. 调整字号阈值

| 层级 | 旧阈值 | 新阈值 |
|------|--------|--------|
| 一级标题 | ≥ 1.3 | **≥ 1.4** |
| 二级标题 | 1.1-1.3 | **1.1-1.4** |
| 三级标题 | 1.0-1.3 | **1.0-1.1** |

**原因**：
- 提高一级标题门槛，避免 "一、" 格式被误判为一级
- 缩小三级标题范围，使更多标题落入二级

#### 3. 当前判定规则

```
标题判定流程：
1. 计算字号比例 = 实际字号 / 基准字号
2. 获取 body_x0（紧跟正文第一行的左边界）
3. 计算 x0 差距 = |当前块x0 - body_x0|

一级标题：font_ratio >= 1.4 
         AND x0差距 > 10点
         AND 垂直留白足够
         AND 短文本(<50字)
         AND 无句号

二级标题：1.1 <= font_ratio < 1.4
         AND x0差距 > 10点
         AND 留白足够
         AND 短文本(<50字)
         AND 无句号

三级标题：1.0 <= font_ratio < 1.1
         AND x0差距 <= 10点
         AND 留白足够
         AND 短文本(<40字)
         AND 无句号
```

#### 4. 文件变更
- 修改：`scripts/readers/pdf_reader.py`
  - 移除 `LEVEL3_PATTERNS` 常量
  - 简化 `_estimate_heading_level` 方法
  - 简化 `_classify_block` 方法
  - 更新文档字符串

## 2026-05-04 更新：修复 EPUB 跨文件内容提取缺陷

- EPUB 中"01 简单"大章节实际包含 17 个连续 HTML 文件（text00007.html 到 text00023.html）

## 2026-05-04 更新：修复 PDF 输出 Markdown 标题层级标记

### 问题描述
PDF 分章节输出的 Markdown 文件中，标题没有使用 `#` 标记标识层级（如 `## 二级标题`、`### 三级标题`）。

### 根因分析
1. `_build_chapters` 函数创建 `ContentBlock` 时没有设置 `level` 属性（默认为 0）
2. `book_processor.py` 中的 `_mark_heading_level` 函数根据 `block.level` 添加 `#` 标记，但 level 始终为 0

### 解决方案
1. **修改 `_build_chapters` 函数**：
   - 为已识别的标题设置正确的层级（1、2、3）
   - 为未识别但可能是标题的内容块根据字号估算层级

2. **新增 `_estimate_heading_level` 方法**：
   - ≥ 1.3 倍基准字号 → 一级标题 (`#`)
   - 1.1 - 1.3 倍 → 二级标题 (`##`)
   - 1.0 - 1.1 倍 → 三级标题 (`###`)

### 修复效果
- Markdown 文件现在正确使用 `#`、`##`、`###` 标识标题层级
- 便于阅读和后续处理

### 文件变更
- 修改：`scripts/readers/pdf_reader.py` - 添加 `_estimate_heading_level` 方法，修改 `_build_chapters`
- 修改：`scripts/book_processor.py` - 确保正确传递 level 属性

## 2026-05-03 更新：PDF 读取器规则重新设计

### 问题描述
原 PDF 读取器规则存在以下问题：
1. 二级/三级标题区分不清晰
2. "一、" 格式标题层级判定不准确
3. 标题合并规则不完善

### 新规则设计

#### 标题层级判定（基于基准字号）

| 层级 | 字号条件 | x0 条件 | 默认是否正文 |
|------|---------|---------|-------------|
| **一级** | ≥ 基准×1.3 | 与紧跟正文差距大（>10点） | 否 |
| **二级** | 1.1-1.3 倍 | 与紧跟正文差距大（>10点） | **是** |
| **三级** | 1.0-1.3 倍 | 与紧跟正文接近（≤10点） | **是** |

**关键区分点**: 二级/三级通过 **x0 与紧跟正文的位置关系** 区分，而非字号

#### 新增控制参数

```python
reader.read(file_path, level2_as_body=True, level3_as_body=True)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `level2_as_body` | True | 二级标题视为正文（不创建独立章节） |
| `level3_as_body` | True | 三级标题视为正文（不创建独立章节） |

#### 合并规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| 一级 + 二级（无正文） | 合并 | "第1章 - 1.1节" |
| 一级 + 一级（第一个不独立） | 合并 | "序言 - 导言" |
| 一级/二级 + 三级 + 正文 | 三级视为正文 | 正文包含三级标题 |

### 文件变更
- 修改：`scripts/readers/pdf_reader.py` - 重写 `_classify_block` 方法
- 修改：`scripts/readers/pdf_reader.py` - 添加控制参数支持
