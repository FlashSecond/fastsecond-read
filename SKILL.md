# FastSecond Read - 智能书籍分章工具

## 概述

本技能只做一件事：**智能分章节，提取内容**。

支持20+种文件格式，自动识别章节边界，将每个章节保存为排版好的Markdown文件。

**核心能力**:
- **智能分章**：基于版面特征（字号/x0位置）或HTML标签层级自动识别章节边界
- **跨文件合并**：EPUB 同一章节跨多个 HTML 文件时自动合并内容
- **Markdown 层级标记**：PDF/EPUB 输出使用标准 `#` 标记标识标题层级
- **多格式支持**：PDF/EPUB/DOCX/MD/TXT/HTML/JSON/CSV/XML/代码等20+种格式
- **合并输出**：可将分章后的内容合并为单一文件，便于整体阅读

## 分章规则

### PDF 分章规则

#### 标题层级判定（自适应两轮检测算法）

PDF读取器使用**自适应两轮检测算法**，能够自动适应不同书籍的排版设计：

**第一轮：固定比率粗筛**

使用固定字号比率收集候选标题：
- 一级候选：字号 ≥ 基准 × 1.4
- 二级候选：字号 > 基准 × 1.0 且 < 基准 × 1.4
- 三级候选：字号 > 基准 × 1.0 且 < 基准 × 1.4

对收集到的字号进行**去重**。

**第二轮：自适应阈值计算**

取去重后的**前两位最大字号**，计算平均数作为自适应比率：
```
自适应比率 = (第一大字号 + 第二大字号) / 2 / 基准字号
```

**第三轮：标题分类（使用自适应比率）**

| 层级 | 字号条件 | x0 条件 | 留白条件 | 默认是否正文 |
|------|---------|---------|----------|-------------|
| **一级** | ≥ 基准 × 自适应比率 | 远离正文 (>10点) | 垂直留白足够 | 否 |
| **二级** | > 基准 × 1.0 且 < 基准 × 自适应比率 | 远离正文 (>10点) | 水平+垂直留白足够 | **是** |
| **三级** | > 基准 × 1.0 且 < 基准 × 自适应比率 | 接近正文 (≤10点) | 水平+垂直留白足够 | **是** |

**关键区分点**:
- **一级**：字号 ≥ 自适应比率（由前两位最大字号平均计算得出）
- **二级/三级**：字号范围相同（>1.0倍 且 < 自适应比率），通过**x0位置**区分
  - 二级：x0远离正文（>10点）
  - 三级：x0接近正文（≤10点）
- 一级/二级标题：仅需满足无句号、留白足够
- **三级标题**：必须满足短文本(<50字)、无句号、留白足够



#### 控制参数

```python
reader.read(file_path, level2_as_body=True, level3_as_body=True)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `level2_as_body` | True | 二级标题视为正文（不创建独立章节） |
| `level3_as_body` | True | 三级标题视为正文（不创建独立章节） |

#### 合并规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| 一级 + 二级（无正文） | 合并 | "第1章 - 1.1节" |
| 一级 + 一级（第一个不独立） | 合并 | "序言 - 导言" |
| 一级/二级 + 三级 + 正文 | 三级视为正文 | 正文包含三级标题 |

### EPUB 分章规则

#### 标题层级判定

| 标签 | 层级 | 默认是否正文 |
|------|------|-------------|
| **h1** | 一级 | 否 |
| **h2** | 二级 | **是** |
| **h3-h6** | 三级 | **是** |

#### 跨文件内容提取

EPUB 中一个大章节可能跨多个 HTML 文件（如 text00007.html 到 text00023.html）。

**处理逻辑**:
1. 根据文件名序号识别连续 HTML 文件
2. 通过标题匹配判断章节边界
3. 将同一章节下的所有 HTML 内容合并

#### 层级验证（2026-05-05 新增）

针对某些EPUB中章节标题和小节使用相同标签级别（如都用`<h1>`）的问题，增加层级验证机制：

**验证规则**:
| 规则 | 说明 | 实现 |
|-----|------|------|
| **章节标题模式识别** | 优先匹配"第X章"格式作为主章节 | 正则匹配 `第[一二三四五六七八九十百千\d]+章` |
| **小节标题识别** | 无"第X章"前缀的一级标题视为小节 | 检查与上一个章节标题的距离 |
| **内容长度阈值** | 小于1000字的"章节"自动合并 | `MIN_SUBSTANTIAL_CHAPTER = 1000` |
| **层级连续性检查** | 检查连续标题的层级关系 | `_validate_heading_hierarchy()` |

**合并逻辑**:
1. 首先验证所有标题的层级关系
2. 标记真正章节标题（含"第X章"）和小节标题
3. 构建章节时跳过小节标题的独立创建
4. 最后将小节内容合并到所属章节

**示例**：《刻意练习》第9章
```
修复前（错误拆分）:
├── 69_第9章　用刻意练习创造全新的世界.md (1.64 KB)
├── 70_用刻意练习原则教物理.md          (7.37 KB)  ← 错误地成为独立章节
├── 71_刻意练习的前景.md                (16.47 KB) ← 错误地成为独立章节
└── 72_创造全新的世界.md                (8.65 KB)  ← 错误地成为独立章节

修复后（正确合并）:
└── 第9章　用刻意练习创造全新的世界.md  (34.27 KB) ← 合并后的完整章节
    ├── 第9章　用刻意练习创造全新的世界
    ├── 用刻意练习原则教物理
    ├── 刻意练习的前景
    └── 创造全新的世界
```

#### 合并规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| h1 + h2（无正文） | 合并 | "第1章 - 1.1节" |
| h1 + h1（第一个不独立） | 合并 | "序言 - 导言" |
| h1/h2 + h3-h6 + 正文 | h3-h6 视为正文 | h1/h2 独立，正文包含 h3-h6 |
| h1(章节) + h1(小节) + 正文 | 小节合并到章节 | "第9章"包含"用刻意练习原则教物理"等小节 |

### 通用过滤规则

| 层级 | 字数限制 | 句末标点 | 关键词过滤 |
|------|---------|---------|-----------|
| **一级标题** | 无限制 | 不以 "。" 结尾 | 适用 |
| **二级标题** | 无限制 | 不以 "。" 结尾 | 适用 |
| **三级标题** | **< 50字** | 不以 "。" 结尾 | 适用 |

- **非章节关键词过滤**: 版权、目录、contents、赞誉、推荐序、译者序、前言、序言、致谢、简介、介绍、导言、引言、后记、附录、参考文献、索引、关于作者、关于本书、献词、扉页、封面

## 使用建议

### 分章后如何使用

本技能仅提供**分章功能**，将书籍拆分为结构化的Markdown文件。分章后的文件可用于：

1. **直接阅读**：每个章节独立成文件，便于单独查阅
2. **AI分析**：将分章后的文件作为输入，使用AI工具进行深度分析
3. **知识管理**：导入Obsidian、Notion等知识管理工具

### AI分析建议

如需对分章后的内容进行AI深度分析，建议：
- 将章节内容发送给AI助手
- 要求生成：章节概述、核心概念、案例分析、思维导图等
- 或使用专门的AI分析工具处理分章后的Markdown文件
