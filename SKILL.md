# FastSecond Read - 智能书籍分章工具

## 概述

本技能只做一件事：**智能分章节，提取内容**。

支持20+种文件格式，自动识别章节边界，将每个章节保存为排版好的Markdown文件。之后由AI模型直接读取分析，生成总结输出。

**核心能力**：
- **智能分章**：基于版面特征（位置/字号/样式）或标题层级自动识别章节边界
- **多格式支持**：PDF/EPUB/DOCX/MD/JSON/CSV/XML/代码等20+种格式
- **排版输出**：每个章节保存为排版好的MD文件，包含标题、元信息、正文内容

---

## 处理流程

```
用户说"提炼这本书"
    ↓
Step 1: 检查是否已分章（MD文件是否存在）
    ├── 不存在 → 调用 book_processor.py 分章
    └── 存在 → 继续
    ↓
Step 2: 【关键】对每个章节进行分析
    ├── 读取 01_章节.md
    ├── AI 分析 content 内容
    ├── 生成 章节分析.md（包含4个部分）
    └── 保存到 第01章_标题/ 目录
    ↓
Step 3: （可选）生成全书总结
    └── 00_全书总结.md
```

**不要只做 Step 1！必须执行 Step 2 的章节分析！**

## 使用方式

### 方式一：整书批量处理（推荐）

用户说 **"提炼这本书"** 或 **"总结全书"** 时：

**AI处理流程**：
1. **检查文件状态**：查看输出目录是否已有分章的 MD 文件
2. **如未分章**：先调用 `book_processor.py` 进行分章，生成 MD 文件
3. **遍历分析**：按顺序读取每个章节的 MD 文件，AI 分析内容
4. **批量生成**：为每个章节生成综合分析 MD 输出文件
5. **汇总整合**：可选生成全书总结

### 方式二：单章分析

针对特定章节深入分析：
- "提炼第3章" → 读取 `03_章节标题.md`，AI 分析生成综合分析
- "分析第5章的案例" → 只读取第5章 MD，只生成案例分析部分

## AI 执行检查清单

当用户要求"提炼/总结/分析"书籍时，AI 必须：

- [ ] **检查分章状态**：查看分章后的 MD 文件是否存在
- [ ] **如未分章**：先执行 `book_processor.py` 生成分章 MD
- [ ] **遍历每个章节**：读取每个 `XX_章节标题.md` 文件
- [ ] **AI 分析内容**：基于 MD 内容进行实际分析
- [ ] **生成综合文件**：每个章节生成 1 个 `章节分析.md` 文件（包含4个部分）
- [ ] **验证输出**：确认每个章节文件夹都有 1 个完整的 MD 文件

**常见错误**：
- ❌ 只生成了分章 MD，没有生成分析 MD 文件
- ❌ 只分析了部分章节，遗漏了其他章节
- ❌ 生成的 MD 文件是空的或只有模板，没有实际分析内容
- ❌ 生成的 MD 文件缺少4个部分中的某些部分

## 命令速查

```bash
# 分章（生成排版好的MD文件）
python book_processor.py "书籍.epub"

# 准备分析任务（生成提示词）
python ai_analyzer.py "分章MD目录"

# 分析指定范围章节
python ai_analyzer.py "分章MD目录" --range 1-10

# 全自动（分章+分析）
python auto_process.py "书籍.epub"

# 验证结果
python verify_analysis.py "分析结果目录"
```
