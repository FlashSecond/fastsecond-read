# FastSecond Read - 智能书籍分章工具

## 概述

本技能只做一件事：**智能分章节，提取内容**。

支持20+种文件格式，自动识别章节边界，将每个章节保存为排版好的Markdown文件。

**核心能力**:
- **智能分章**：基于版面特征（字号/x0位置）或HTML标签层级自动识别章节边界
- **跨文件合并**：EPUB 同一章节跨多个 HTML 文件时自动合并内容
- **Markdown 层级标记**：PDF/EPUB 输出使用标准 `#` 标记标识标题层级
- **多格式支持**：PDF/EPUB/DOCX/MD/TXT/HTML/JSON/CSV/XML/代码等20+种格式

---

## 使用方式

### 分章命令

```bash
# 分章书籍/文档
python book_processor.py "书籍.pdf"
```

**示例**:
```bash
python book_processor.py "D:/Mylibrary/消费心理学.pdf"
```

**输出位置**: `D:/Mylibrary/书籍总结/消费心理学/`

## 分章规则

### PDF 分章规则

#### 标题层级判定（基于基准字号 + x0位置）

| 层级 | 字号条件 | x0 条件 | 留白条件 | 默认是否正文 |
|------|---------|---------|----------|-------------|
| **一级** | ≥ 基准×1.4 | 远离正文 (>10点) | 垂直留白足够 | 否 |
| **二级** | 1.0-1.4 倍 | 远离正文 (>10点) | 水平+垂直留白足够 | **是** |
| **三级** | 1.0-1.4 倍 | 接近正文 (≤10点) | 水平+垂直留白足够 | **是** |

**关键区分点**:
- 一级通过 **字号≥1.4** 区分（字号门槛最高）
- 二级/三级通过 **x0与正文位置关系** 区分（远离vs接近）
- 所有标题必须满足：短文本(<50字)、无句号、留白足够



#### 控制参数

```python
reader.read(file_path, level2_as_body=True, level3_as_body=True)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `level2_as_body` | True | 二级标题视为正文（不创建独立章节） |
| `level3_as_body` | True | 三级标题视为正文（不创建独立章节） |

#### 合并规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| 一级 + 二级（无正文） | 合并 | "第1章 - 1.1节" |
| 一级 + 一级（第一个不独立） | 合并 | "序言 - 导言" |
| 一级/二级 + 三级 + 正文 | 三级视为正文 | 正文包含三级标题 |

### EPUB 分章规则

#### 标题层级判定

| 标签 | 层级 | 默认是否正文 |
|------|------|-------------|
| **h1** | 一级 | 否 |
| **h2** | 二级 | **是** |
| **h3-h6** | 三级 | **是** |

#### 跨文件内容提取

EPUB 中一个大章节可能跨多个 HTML 文件（如 text00007.html 到 text00023.html）。

**处理逻辑**:
1. 根据文件名序号识别连续 HTML 文件
2. 通过标题匹配判断章节边界
3. 将同一章节下的所有 HTML 内容合并

#### 合并规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| h1 + h2（无正文） | 合并 | "第1章 - 1.1节" |
| h1 + h1（第一个不独立） | 合并 | "序言 - 导言" |
| h1/h2 + h3-h6 + 正文 | h3-h6 视为正文 | h1/h2 独立，正文包含 h3-h6 |

### 通用过滤规则

- **标题字数**: < 50字
- **句末标点**: 不以 "。" 结尾
- **非章节关键词过滤**: 版权、目录、contents、赞誉、推荐序、译者序、前言、序言、致谢、简介、介绍、导言、引言、后记、附录、参考文献、索引、关于作者、关于本书、献词、扉页、封面
