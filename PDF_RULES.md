# PDF/EPUB 分章规则详解

## 通用过滤规则

### 1. 标题字数限制
- **规则**: 标题字数 < 50字
- **目的**: 排除过长的正文句子

### 2. 短文本检测
- **规则**: 通过字数限制控制
- **目的**: 区分标题与正文段落

### 3. 句末标点检测
- **规则**: 不以句号 "。" 结尾
- **目的**: 正文通常有句号，标题通常无

### 4. 非章节关键词过滤
- **规则**: 排除特定关键词
- **关键词列表**:
  - 版权、目录、contents、赞誉、推荐序、译者序
  - 前言、序言、致谢、简介、介绍、导言、引言
  - 后记、附录、参考文献、索引
  - 关于作者、关于本书、献词、扉页、封面

## PDF 分章规则

### 基础数据收集
1. 获取行宽度（页面内容区域）
2. 收集整个文档所有文本块字号，统计频率最高的字号，视为正文基准字号
3. 提取所有文本块，每个文本块包含：文本内容、包围盒坐标(x0,y0,x1,y1)、字体名称、字体大小

### 标题与正文的区分

| 规则 | 条件 | 说明 |
|------|------|------|
| **x0位置** | 三级标题x0接近正文；一级/二级标题x0远离正文 | 通过左边界区分 |
| **留白** | 句前+句后留白 > 行宽一半 | 通过x1-x0计算 |
| **短文本** | 字数 < 50 | 排除长段落 |
| **无句号** | 不以"。"结尾 | 标题通常无句号 |

### 标题层级判定（基于基准字号 + x0位置）

| 层级 | 字号条件 | x0条件 | 是否视为正文 |
|------|---------|--------|-------------|
| **一级标题** | ≥ 基准×1.4 | 与紧跟正文差距大（>10点） | 否（始终为章节） |
| **二级标题** | 1.0-1.4 倍 | 与紧跟正文差距大（>10点） | **可配置**（默认是） |
| **三级标题** | 1.0-1.4 倍 | 与紧跟正文接近（≤10点） | **可配置**（默认是） |

**关键区分点**：
- **一级**：字号门槛最高（≥1.4），通过字号区分
- **二级/三级**：字号范围相同（1.0-1.4），通过**x0与紧跟正文的位置关系**区分
  - 二级：x0远离正文（>10点）
  - 三级：x0接近正文（≤10点）
- **二级、三级默认为是正文**，可通过参数控制是否视为独立章节

### 控制参数

```python
reader.read(file_path, level2_as_body=True, level3_as_body=True)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `level2_as_body` | True | 二级标题视为正文（不创建独立章节） |
| `level3_as_body` | True | 三级标题视为正文（不创建独立章节） |

**使用示例**：
```python
# 默认：二级、三级都视为正文
reader.read("book.pdf")

# 二级视为章节，三级视为正文
reader.read("book.pdf", level2_as_body=False, level3_as_body=True)

# 二级、三级都视为章节
reader.read("book.pdf", level2_as_body=False, level3_as_body=False)
```

### 独立标题判定

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| 标题后紧跟正文 | **独立标题** | 一级+正文=独立一级 |
| 一级+二级+正文 | **合并为独立标题** | "第1章 - 1.1节" |
| 一级+一级+正文 | **合并为独立标题** | "序言 - 导言" |
| 一级/二级+三级+正文 | **三级视为正文** | 一级/二级独立，正文包含三级 |

## EPUB 分章规则

### 基础数据收集
1. 解析EPUB中的所有HTML文件
2. 提取h1-h6标签作为标题候选

### 标题层级判定
| 标签 | 层级 | 是否视为正文 |
|------|------|-------------|
| **h1** | 一级标题 | 否（始终为章节） |
| **h2** | 二级标题 | **可配置**（默认是） |
| **h3-h6** | 三级标题 | **可配置**（默认是） |

**控制参数**（PDF & EPUB通用）：
```python
# PDF
reader.read(file_path, level2_as_body=True, level3_as_body=True)

# EPUB
reader.read(file_path, level2_as_body=True, level3_as_body=True)
```
- `level2_as_body`: 二级标题视为正文（默认True）
- `level3_as_body`: 三级标题视为正文（默认True）

### 独立标题判定
- **独立标题**: 后面紧跟正文内容（p, div等非标题标签）
- **非独立标题**: 后面是更低层级标题（需合并）

### 合并规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| h1 + h2（无正文） | 合并为一个独立标题 | "第1章 - 1.1节" |
| h1 + h1（第一个不独立） | 合并为一个独立标题 | "序言 - 导言" |

### 三级标题特殊处理（EPUB）

**规则**: h1/h2 + h3-h6 + 正文 → h3-h6视为正文

**说明**:
- 如果h1或h2后面紧跟着h3-h6，且h3-h6后面有正文
- 则h3-h6被视为正文内容的一部分，不归入章节标题
- h1/h2标题独立，正文包含h3-h6及其后的内容

## 更新历史

- **2026-05-05**: PDF规则最终优化
  - 统一二级/三级字号范围为 1.0-1.4
  - 完全通过 x0 位置区分二级（远离正文）和三级（接近正文）
  - 提高一级标题门槛至 ≥1.4
- **2026-05-03**: PDF规则重新设计
  - 通过**x0与紧跟正文的位置关系**区分二级和三级
  - 关键修复：每个标题使用自己后面紧跟的正文x0来判断，而不是文档开头的正文x0
- **2026-05-03**: 新增控制参数 `level2_as_body` 和 `level3_as_body`
  - PDF和EPUB都支持
  - 可配置二级、三级标题是否视为正文（默认都视为正文）
