# PDF/EPUB 分章规则详解

## 通用过滤规则

### 1. 标题字数限制
- **一级标题**: 无字数限制
- **二级标题**: 无字数限制
- **三级标题**: < 50字（仅对三级标题生效，防止正文段落被误判）
- **目的**: 排除过长的正文句子

### 2. 句末标点检测
- **规则**: 不以句号 "。" 结尾
- **目的**: 正文通常有句号，标题通常无

### 3. 非章节关键词过滤
- **规则**: 排除特定关键词
- **关键词列表**:
  - 版权、目录、contents、赞誉、推荐序、译者序
  - 前言、序言、致谢、简介、介绍、导言、引言
  - 后记、附录、参考文献、索引
  - 关于作者、关于本书、献词、扉页、封面

## PDF 分章规则

### 基础数据收集
1. 获取行宽度（页面内容区域）
2. 收集整个文档所有文本块字号，统计频率最高的字号，视为正文基准字号
3. 提取所有文本块，每个文本块包含：文本内容、包围盒坐标(x0,y0,x1,y1)、字体名称、字体大小

### 标题层级判定（自适应两轮检测算法）

PDF读取器使用**自适应两轮检测算法**来识别标题层级，能够自动适应不同书籍的排版设计。

#### 第一轮：固定比率粗筛

使用**固定字号比率**收集符合各级标题条件的字号：

| 目标层级 | 字号条件 | 其他条件 |
|---------|---------|---------|
| **一级候选** | ≥ 基准字号 × 1.4 | 无句号 + 留白足够 |
| **二级候选** | > 基准字号 × 1.0 且 < 基准字号 × 1.4 | 无句号 + 留白足够 |
| **三级候选** | > 基准字号 × 1.0 且 < 基准字号 × 1.4 | 短文本(<50字) + 无句号 + 留白足够 |

收集所有通过筛选的候选标题字号，进行**去重**。

#### 第二轮：自适应阈值计算

```
算法步骤：
1. 统计候选标题字号的频率（不去重，每个候选都计数）
2. 去掉频率最高的字号（通常是正文基准字号）
3. 计算剩余字号的期望值（加权平均）作为自适应阈值：
   
   期望值 = Σ(字号 × 频率) / Σ频率
   
4. 使用期望值作为一级标题阈值，期望值的80%作为二级标题阈值
```

**示例**：
- 基准字号 = 12pt
- 候选字号及频率：18pt(12次), 15pt(45次), 13.5pt(8次), 12.5pt(120次)
- 去掉频率最高的12.5pt（120次，正文基准）
- 剩余加权平均 = (18×12 + 15×45 + 13.5×8) / (12+45+8) = 1035 / 65 = **15.92pt**
- 一级阈值 = 15.92pt，二级阈值 = 15.92 × 0.8 = 12.74pt

#### 第三轮：标题分类（使用自适应阈值）

使用计算出的**期望值**作为自适应阈值重新分类所有文本块：

| 层级 | 字号条件 | x0条件 | 其他条件 |
|------|---------|--------|---------|
| **一级标题** | ≥ 期望值 | 垂直留白足够 | 无句号 + (居中 或 x0远离正文) |
| **二级标题** | > 基准 × 1.0 且 < 期望值 | 留白足够 + x0远离正文 | 无句号 |
| **三级标题** | > 基准 × 1.0 且 < 期望值 | 留白足够 + x0接近正文 | 短文本(<50字) + 无句号 |

**关键区分点**：
- **一级/二级标题**：无字数限制，仅需满足无句号、留白足够
- **三级标题**：必须满足 < 50字，防止正文段落被误判为标题

**关键区分点**：
- **一级**：字号 ≥ 期望值（去掉最高频字号后的加权平均）
- **二级/三级**：字号范围相同（>1.0倍 且 < 期望值），通过**x0位置**区分
  - 二级：x0远离正文（与紧跟正文的x0差距 > 10点）
  - 三级：x0接近正文（与紧跟正文的x0差距 ≤ 10点）
- **字数限制**：仅对三级标题生效（<50字），一级/二级标题无字数限制

### 控制参数

```python
reader.read(file_path, level2_as_body=True, level3_as_body=True)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `level2_as_body` | True | 二级标题视为正文（不创建独立章节） |
| `level3_as_body` | True | 三级标题视为正文（不创建独立章节） |

**使用示例**：
```python
# 默认：二级、三级都视为正文
reader.read("book.pdf")

# 二级视为章节，三级视为正文
reader.read("book.pdf", level2_as_body=False, level3_as_body=True)

# 二级、三级都视为章节
reader.read("book.pdf", level2_as_body=False, level3_as_body=False)
```

### 独立标题判定

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| 标题后紧跟正文 | **独立标题** | 一级+正文=独立一级 |
| 一级+二级+正文 | **合并为独立标题** | "第1章 - 1.1节" |
| 一级+一级+正文 | **合并为独立标题** | "序言 - 导言" |
| 一级/二级+三级+正文 | **三级视为正文** | 一级/二级独立，正文包含三级 |

## EPUB 分章规则

### 基础数据收集
1. 解析EPUB中的所有HTML文件
2. 提取h1-h6标签作为标题候选

### 标题层级判定
| 标签 | 层级 | 是否视为正文 |
|------|------|-------------|
| **h1** | 一级标题 | 否（始终为章节） |
| **h2** | 二级标题 | **可配置**（默认是） |
| **h3-h6** | 三级标题 | **可配置**（默认是） |

**控制参数**（PDF & EPUB通用）：
```python
# PDF
reader.read(file_path, level2_as_body=True, level3_as_body=True)

# EPUB
reader.read(file_path, level2_as_body=True, level3_as_body=True)
```
- `level2_as_body`: 二级标题视为正文（默认True）
- `level3_as_body`: 三级标题视为正文（默认True）

### 独立标题判定
- **独立标题**: 后面紧跟正文内容（p, div等非标题标签）
- **非独立标题**: 后面是更低层级标题（需合并）

### 合并规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| h1 + h2（无正文） | 合并为一个独立标题 | "第1章 - 1.1节" |
| h1 + h1（第一个不独立） | 合并为一个独立标题 | "序言 - 导言" |

### 三级标题特殊处理（EPUB）

**规则**: h1/h2 + h3-h6 + 正文 → h3-h6视为正文

**说明**:
- 如果h1或h2后面紧跟着h3-h6，且h3-h6后面有正文
- 则h3-h6被视为正文内容的一部分，不归入章节标题
- h1/h2标题独立，正文包含h3-h6及其后的内容
